# 策略推荐方案：日内事件驱动 + 实时监控

> 这是一套基于实时数据的日内事件驱动量化策略。监测盘口异动，事件触发即交易。

---

## 重要：回测与实盘的差距

在开始之前，先把这个说清楚：

| | 多因子日频策略 | 本方案（事件驱动） |
|------|--------------|------------------|
| **数据需求** | 日线（OHLCV），免费易得 | Tick 级 / 分钟级 + Level-2 盘口数据 |
| **回测真实性** | 高，日线回测 ≈ 实盘 | 中低，很难模拟真实的盘口排队和滑点 |
| **回测能测什么** | 因子有效性、组合收益 | 信号触发逻辑对不对、大致的胜率盈亏比 |
| **回测测不了什么** | — | 盘口排队能不能买到、真实滑点多大、抢单速度 |
| **验证方式** | 回测有信心后直接实盘 | 回测只能验证逻辑，**必须经过模拟盘充分验证**才能上实盘 |

**大白话**：这种策略回测只能证明"思路对"，不能证明"能赚钱"。真正的验证靠 QMT 模拟盘跑。

---

## 一、策略整体框架

```
盘中实时运行（QMT 服务器上，或本地 QMT 客户端）：

Level-2 行情流 → 逐笔成交 / 盘口变化
    → 事件检测引擎（同时监听 6 类事件）
        → 某事件触发 → 检查风控条件 → 自动下单
```

---

## 二、事件检测清单（策略的"耳朵"，听什么）

### 事件 1：跌停板翘板（你举的例子）

**检测逻辑**：

1. 股票触及跌停价（或跌停板上封单量巨大）
2. 连续出现大买单（单笔 > 50 万 or 占 1 分钟成交额 30% 以上）
3. 跌停板上的封单量在 3 分钟内减少了 40% 以上
4. → **触发买入信号**

**参数**：
| 参数 | 值 | 说明 |
|------|-----|------|
| 监控标的 | 股票池内所有股票 | 不监控全市场（数据量太大） |
| 跌停确认 | 价格 ≤ 跌停价 × 1.002 | 允许 0.2% 容差 |
| 大买单阈值 | 单笔 ≥ 30 万元 | 可根据资金量调整 |
| 封单衰减 | 3 分钟内封单减少 ≥ 30% | 说明有人在吃封单 |
| 确认窗口 | 5 分钟内 | 条件持续满足才触发 |

### 事件 2：大资金持续流入

**检测逻辑**：

1. 分钟级别监测：连续 5 分钟，每分钟主动买入额 > 主动卖出额 × 1.5
2. 且期间价格涨幅 < 2%（说明还没涨起来，大资金在悄悄吸筹）
3. → **触发买入信号**

**参数**：
| 参数 | 值 |
|------|-----|
| 连续分钟数 | ≥ 5 分钟 |
| 主动买 / 主动卖比例 | > 1.5 |
| 期间涨幅上限 | < 2%（避免追高） |
| 成交量放大 | 当前分钟量 > 过去 5 日同时段均量 × 2 |

### 事件 3：盘口异动（大单扫货）

**检测逻辑**：

1. 卖一到卖五的挂单总量忽然被吃掉 50% 以上
2. 且买一到买五的挂单量在增加（说明下方有人在垫单）
3. → **触发买入信号**

这个需要 Level-2 的五档盘口数据。

**参数**：
| 参数 | 值 |
|------|-----|
| 卖盘衰减比例 | ≥ 50% |
| 时间窗口 | 30 秒内 |
| 买盘增量 | ≥ 20% |

### 事件 4：涨停板封板（打板）

**检测逻辑**：

1. 股票接近涨停（涨幅 > 9.5%）
2. 封板瞬间买一封单量 > 5000 手
3. 且封板前 5 分钟换手率 > 3%（说明充分换手，封板质量高）
4. → **触发买入信号（排队打板）**

### 事件 5：开盘放量跳空

**检测逻辑**：

1. 开盘价较昨日收盘跳空高开 ≥ 2%
2. 开盘 5 分钟内成交额超过昨日全天成交额的 15%
3. 且开盘后价格没有回补缺口（最低价 > 开盘价 × 0.99）
4. → **触发买入信号（强势突破确认）**

### 事件 6：午后拉升

**检测逻辑**：

1. 上午涨幅在 -1% ~ +1% 之间（横盘）
2. 13:00 ~ 14:00 之间突然拉升，15 分钟内涨幅 > 3%
3. 且拉升期间成交量 > 同时段均量 × 3
4. → **触发买入信号**

**大白话**：很多机构喜欢午后动手，上午让市场自由博弈，下午看清楚了再拉升。

## 三、卖出逻辑（平仓条件）

### 主动平仓（信号触发）

| 事件 | 逻辑 |
|------|------|
| **止损** | 买入后亏损达 -5%，立即卖出 |
| **止盈** | 买入后盈利达 +8%，卖出 |
| **时限止盈** | 持仓满 5 个交易日仍未触发其他卖出条件，卖出（事件驱动的机会有时效性） |
| **反向信号** | 大资金持续流出（连续 3 分钟主动卖出 > 主动买入 × 1.5） |

### 尾盘强制平仓

如果当日买入的标的到 14:55 仍未涨停且当日盈利 < 2%，则卖出。日内事件驱动的逻辑是"当天要有反应"，没反应的就不留了。

## 四、仓位与资金管理

| 参数 | 值 | 说明 |
|------|-----|------|
| 总资金 | 200,000 | |
| 单笔交易最大资金 | 20,000（10%） | 每次触发只买 2 万 |
| 同时持仓最大数量 | 5 笔 | 最多同时持有 5 个机会 |
| 每日最大交易次数 | 10 次 | 限制过度交易 |
| 每日最大亏损额 | 5,000 元（2.5%） | 当日亏损到此金额停止交易 |
| 单股重复买入冷却 | 同一只股票卖出后 30 分钟内不再买入 | 避免反复被收割 |

## 五、全局风控

| 层级 | 措施 |
|------|------|
| **账户日亏损熔断** | 当日亏损 ≥ 5,000 元 → 暂停当日所有交易 |
| **账户周亏损熔断** | 当周累计亏损 ≥ 12,000 元 → 暂停当周剩余交易日 |
| **连续亏损熔断** | 连续 5 笔交易亏损 → 暂停 1 小时，重新评估 |
| **大盘环境** | 上证指数当日跌幅 > 2% → 只平仓，不开新仓 |
| **跌停板数量** | 全市场跌停家数 > 50 家 → 暂停开仓（市场恐慌） |
| **黑名单** | 同一只股票连续止损 2 次 → 该股票进入黑名单 20 个交易日 |

## 六、回测能做什么、不能做什么

### 回测能做

- 用 **1 分钟线数据** 模拟事件检测逻辑
- 验证事件触发后的胜率和盈亏比
- 验证止损止盈参数是否合理
- 输出：事件触发统计、胜率、盈亏比、收益曲线

### 回测做不了

- 模拟真实的盘口排队（能不能买到）
- 模拟真实的滑点（实际成交价和信号价的差异）
- 模拟 Level-2 盘口数据（回测通常拿不到历史 L2 数据）
- 评估抢单速度对结果的影响

### 所以回测的意义

回测能告诉你：**"这个事件本身的逻辑靠不靠谱"**。比如"跌停翘板"在过去 4 年里触发了多少次、成功率多少、盈亏比如何。如果连事件本身的统计都很难看，那实盘肯定也不行。

但如果回测看起来不错，**不代表实盘就一定能赚钱**——实盘的盘口执行是另一个变量。这需要 QMT 模拟盘验证。

## 八、大白话总结

```
这套策略像个坐在交易终端前的机器人：

1. 眼睛盯着股票池里每只股票的每一笔成交和盘口变化
2. 耳朵听着 6 种特定的事件信号
3. 听到信号 → 检查风控没拦着 → 立刻下单
4. 每天最多交易 10 次，亏到 5000 块就停手

不是"每天算个分、明天买一篮子"。
而是"哪里有异动，哪里就有反应"。
```
